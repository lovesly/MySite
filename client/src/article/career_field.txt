为什么会成为一名前端。 与其说是选择， 不如说是冥冥之中， 无数巧合。 
为什么会是前端呢， 我在思考这个问题的时候， 脑海里浮现的是星爷在殴打菩提老祖， 质问他为什么非要变葡萄。 
我身边并没有人可以拿来殴打， 只好自己来回答。 

时间回到4年以前， 彼时我以未来的电子/自动化工程师自居， 结果发现硬件工作机会寥寥， 招聘会是来了不少公司没错， 不过一是不少工作要求身家清白，根正苗绿。 二是工作要求和我学的严重不符。 这时候首先想到的就是抽中介俩大嘴巴， 完全不是描绘中的 EE 很好找工作的样子。 好在我们专业和 CS 联系紧密， 有些课都是一样的， 转起行来相当自然。 

我的启蒙读物是一本 PHP + MySQL， 加上一本 JQuery/HTML/CSS, 课上学过数据库， 算法， 图像， 密码。 总算有了系统一点的认识。 人生第一个面试就差一点拿下， 4 轮里有 3 轮相当顺利， 事情坏就坏在一句话上， 半年以前上的数据库课程， 那时候各种概念如数家珍，写个语句手到擒来。 不过考完试就就忘得七七八八， 可是简历里写的是 “精通”。 可把我问惨了， 其实这工作根本就不需要多么高深的数据库知识， 增删改查而已。 面试官说：“我知道啊， 但是你这里写着。。。” 真是好好给我上了一课， 此后再也不敢夸大其词， 我应该说略懂。 

第一次便和目标近在咫尺， 让我大大低谷了后面的苦难。 实习希望你有工作经验， 全职希望你有实习经验， 但是我屁的经验也没有。 反正也不知道会做什么， 所以我什么职位都投， 面过 PHP 的职位， 面过测试的职位， 面过安全的职位， 面过前端的职位， 面过 Java 的职位。 好多次倒在最后一轮， 有实力不济， 也有运气使然， 还有莫名其妙的， 最让我印象深刻的是校招上给你留作业的， 一周时间， 用 Node.js 做个后台 API 返回数据到页面上。 彼时我连 Node.js 是什么都不知道， 不过机会总不能放过， 搞搞看。 我打开 Node的官网， 看得懵懵懂懂， 唉这有个链接， npm， npm 又是什么东西， 点开看看， 我靠出现一堆链接， 我记得头几个是 Gulp， Bower， Browserify 之类的, 这时候我已经得拿手撑着额头， 彼时我对前端的印象还是三大件， 迷宫一样的各种包， 无异于告诉一个天圆地方的兄弟， 你知道银河系吗。 后来总算做出来， 面试也很顺利， 最后一轮和颜悦色问我你想做前端啊，还是后端啊。 我想了两分钟， 说都可以， 然后悲剧了。 

回首往事， 常常想起那个一亩三分地的兄弟， 论坛里几家欢喜几家愁， 有手握无数offer求助去哪个好的， 也有重压之下上来吐苦水的。 我记得有一个帖子， 标题是“为什么我leetcode刷了5遍还是没offer”， 十分惊悚， 脑子里直接就蹦出那句， 真的猛士敢于直面惨淡的人生。 5遍啊， 简直是猛将兄。 那时候 leetcode 有大概 230 道题。 时隔3年， leetcode 题目数量 800 有余。 怒刷5遍的猛将兄恐怕已是绝唱。 不过让我记忆犹新的倒不是猛将兄， 而是一个留言， 他说：“不要怂， 继续干”。 

总之后来阴差阳错， 选了家大公司， 刚好写前端。 前端好玩吗， 还是挺好玩的， 在我看来唯二的缺点之一是在飞速发展的阶段， 知识的淘汰率太高， 日新月异的新轮子， 谁也不知道 2 年以后你的 2 年经验还值不值 2 年， 这也是为什么我特别认同某位大佬说的话， 你首先是工程师， 然后才是前端工程师。 另一个缺点就是 IE 了， 回顾历史， 让我们翻开《高级程序设计》一小半儿都是如何安抚孤傲的 IE。 每当看到如何兼容 IE 的问题， 简直是热血上涌， 恨不得拍案而起，大吼一声，全世界的前端联合起来， 反抗 IE 暴政。 不过前端也不是没好处， 论坛为谁是最好的语言打架的时候， 前端很少会被殃及池鱼， PHP， Java， Python， C/C++ 打得头破血流， 谁都没想起来还有一撮儿写JS的。
